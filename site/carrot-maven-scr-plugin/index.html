About

Context

OSGI Declarative Sevices (p 297) is old , tried , and proven technology, now getting a new life (p 187) .

Purpose

this plugin replaces popular felix scr annotations framework components,

including osgi annotations , annotations processor , annotations maven plugin

with new OSGI specification compliant (p 187) counterparts.

Motivation

felix scr annotations framework does not integrate well with eclipse interactive development mode:

  • very slow
  • batch mode only
  • no incremental build
  • generates single monolititc descriptor file
  • obscure / buggy component inheritance

Solution

carrrot scr annotations framework resolves these issues as follows:

  • does it fast
  • support both batch & incremental mode
  • provide incremental/interactive build in eclipse
  • generates individual per component descriptor files
  • features transparent component attributes inheritance

Repository

maven central has latest version of the plugin

M2E Integration

the foregoing assumes that you are already using latest m2e 1.1 or m2e 1.2 or m2e 1.3 maven integration in your eclipse profile

this plugin comes with a companion m2e maven/eclipse connector plugin

available form the following update site location

when both maven plugin and eclipse plugin are properly installed, you should see the following picture in

Eclipse -> Project -> Properties -> Maven -> Lifecycle Mapping

Eclipse Maven Console

to verify incremental build in eclipse, you can activate

Eclipse -> Window -> Show View -> Console -> Maven Console
